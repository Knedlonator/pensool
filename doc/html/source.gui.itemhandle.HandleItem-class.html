Class HandleItem

source code

A HandleItem control is a member of a HandleMenu group. It :

  • appears on the edge of a controlee
  • mouseover appearance and dissappearance, without a click
  • not usually clickable (drag is preferred) ???
  • tracks mouse: moveable along the path of its controlee (without a button press)
  • starts drag (after a press)

Singleton subclasses with different look and different behavior upon:

  • end drag
  • scroll

mouse_exit(*args, **kwargs)

 

Mouse exited item. This is a filtered event from super GuiControl. Assert no button down (not a drag.)

Dynamic orthogonality. A handle menu MIGHT follow the mouse in orthogonal directions, if the handle menu is a TrackingHandleMenu AND we didn't hit a stop (the limit of an edge of the controlee morph.) Exit from item is EITHER in "menu exit" direction OR in "menu change item" direction.
