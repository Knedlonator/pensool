  1  ''' 
  2  Pointer manager: state machine for pointer motion 
  3  The pointer must stop (remain within a few pixels for a short period) 
  4  before mouseover (picking) is done. 
  5   
  6  Alternatively, we could pick at every mouse move, and delay the result 
  7  of the pick until it was confirmed a short time later. 
  8  But that requires efficient picking so as not to be jerky. 
  9   
 10  Here we use the mouse speed in pixel distance per second (?). 
 11  Using absolute mouse move distances is subject to jitter 
 12  and system busy problems. 
 13   
 14  We might be able to reconstruct a more accurate mouse position (float) 
 15  by averaging with previous mouse position. 
 16  The drawing library is in floating point, why not use mouse position in float? 
 17   
 18  !!! Note that the callback is asynchronous. 
 19  In general, it must be prepared for a race condition. 
 20  For example, the user may have just opened a menu using for example a control key, 
 21  and the callback must not interfere. 
 22   
 23  To test: python -m doctest -v gui/manager/pointer.py 
 24   
 25  Examples: 
 26     
 27    # Normal sequence 
 28    >>> import collections 
 29    >>> import time 
 30    >>> Event = collections.namedtuple('Event', 'x y time') 
 31    >>> foo = Event(1,1, 1) 
 32    >>> bar = Event(1,1, 5) 
 33    >>> register_callback(dummy_callback) 
 34    >>> decide_stopped(foo) 
 35    >>> decide_stopped(bar) 
 36    >>> time.sleep(2) 
 37    callback should have been called. 
 38     
 39    # How to get a callback? 
 40    This is not testable without a doctest having asynchronous or threaded ops, 
 41    such as in Zope?
